# План: Кнопка генерации тестовых данных в Danger Zone

## Обзор задачи

Добавить в настройках приложения (Danger Zone) кнопку для заполнения всех таблиц тестовыми данными за последнюю неделю. Данные должны добавляться к существующим (без очистки).

## Структура базы данных

### Основные таблицы

- `logs` - записи (питание, тренировки, финансы)
- `items` - справочник (витамины, лекарства, травы, косметика, продукты)
- `content` - контент (книги, рецепты)

### Справочные данные

- `categories` - категории
- `tags` - теги
- `units` - единицы измерения
- `accounts` - счета
- `exercises` - упражнения

### Рецепты

- `recipeIngredients` - ингредиенты
- `recipeIngredientItems` - связь рецепт-ингредиент
- `recipeSteps` - шаги приготовления

### Книги

- `books` - книги
- `userBooks` - пользовательские книги
- `authors` - авторы
- `bookAuthors` - связь книга-автор
- `series` - серии
- `genres` - жанры
- `bookGenres` - связь книга-жанр
- `bookQuotes` - цитаты
- `bookReviews` - рецензии

### Цели и привычки

- `goals` - цели
- `habits` - привычки
- `habitLogs` - логи привычек
- `streaks` - серии

### Трекеры

- `sleepLogs` - сон
- `waterLogs` - вода
- `moodLogs` - настроение
- `bodyMeasurements` - измерения тела

### Напоминания

- `reminders` - напоминания
- `reminderLogs` - логи напоминаний

### Шаблоны

- `templates` - шаблоны

### Финансы

- `recurringTransactions` - повторяющиеся транзакции

## План реализации

### Шаг 1: Создать файл seed-данных (src/lib/seed.ts)

Создать модуль с функциями для генерации тестовых данных:

1. **Вспомогательные функции**:
   - `getRandomDateInLastWeek()` - случайная дата за последнюю неделю
   - `getRandomItem<T>(array: T[])` - случайный элемент массива
   - `generateUUID()` - генерация UUID

2. **Генерация справочных данных**:
   - `seedCategories()` - категории для логов, items, контента
   - `seedTags()` - случайные теги
   - `seedUnits()` - единицы измерения
   - `seedAccounts()` - финансовые счета
   - `seedExercises()` - упражнения

3. **Генерация основных данных**:
   - `seedLogs(count: number)` - логи за последнюю неделю (еда, тренировки, финансы)
   - `seedItems(count: number)` - элементы каталога
   - `seedContent(count: number)` - контент (книги, рецепты)

4. **Генерация связанных данных**:
   - `seedRecipes()` - рецепты с ингредиентами и шагами
   - `seedBooks()` - книги с авторами, жанрами, цитатами
   - `seedGoalsAndHabits()` - цели и привычки
   - `seedTrackers()` - данные трекеров за неделю
   - `seedReminders()` - напоминания
   - `seedTemplates()` - шаблоны

### Шаг 2: Обновить DangerZone (src/components/settings/danger-zone.tsx)

Добавить:

1. Импорт функции `seedDatabase` из нового модуля
2. Новую кнопку "Заполнить тестовыми данными"
3. Обработчик с подтверждением
4. Загрузку/уведомление о результате

### Шаг 3: Добавить переводы

Добавить переводы в файлы локализации:

- `dangerZone.seedData` - текст кнопки
- `dangerZone.confirmSeed` - подтверждение
- `dangerZone.seedSuccess` - успех

## Детали генерации данных

### Период: последняя неделю

- Даты: с 2026-02-23 по 2026-03-02
- Текущая дата: 2026-03-02

### Количество записей

- Логи: 15-20 записей (по 5-7 каждого типа)
- Items: 10-15 элементов
- Content: 5-6 записей
- Привычки: 3-5 привычек
- Трекеры: по 5-7 записей каждого типа
- Остальное: по 2-5 записей

### Связанность данных

- Категории создаются first
- Logs ссылаются на существующие categories
- Items ссылаются на categories
- Рецепты ссылаются на ingredients
- Книги ссылаются на authors, genres
- habitLogs ссылаются на habits
