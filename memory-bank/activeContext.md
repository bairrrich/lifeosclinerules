## Активные решения и соображения

### Архитектура книг
- **Book = метаданные** — объективная информация (не зависит от пользователя)
- **UserBook = персональные данные** — статус, прогресс, оценка, цитаты
- **Связь many-to-many** — книга может иметь нескольких авторов
- **Жанры и теги** — отдельно, с возможностью иерархии

### UI паттерны
- Tabs для статусов чтения и типов рецептов
- Card секции для логической группировки полей
- Progress bar для отслеживания чтения
- Badge с цветовой кодировкой для статусов

## Важные паттерны и предпочтения

### Структура маршрутов
```
/books          → список книг
/books/new      → создание
/books/[id]     → просмотр
/books/[id]/edit→ редактирование

/recipes        → список рецептов
/recipes/new    → создание
/recipes/[id]   → просмотр
/recipes/[id]/edit → редактирование
```
