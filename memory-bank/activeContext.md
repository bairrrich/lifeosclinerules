## Активные решения и соображения

### Архитектура данных
- **Log.metadata** — разные типы метаданных для разных типов логов
- **Item.calories/protein/fat/carbs** — пищевая ценность для продуктов
- **Book отдельная таблица** — книги не в content, а в своей таблице

### UI паттерны
- Tabs для статусов чтения и типов рецептов
- Card секции для логической группировки полей
- Combobox для выпадающих списков с поиском
- mounted state для предотвращения hydration mismatch

## Важные паттерны и предпочтения

### Структура маршрутов
```
/books          → список книг
/books/new      → создание
/books/[id]     → просмотр
/books/[id]/edit→ редактирование

/recipes        → список рецептов
/recipes/new    → создание
/recipes/[id]   → просмотр
/recipes/[id]/edit → редактирование
```

### Именование
- Директории: kebab-case (например, `book-form.tsx`)
- Компоненты: PascalCase
- Функции: camelCase

### Работа с данными
- `db.books.count()` возвращает число напрямую
- `db.content.where("type").equals("recipe").count()` для фильтрации
- `log.metadata.calories`, `log.metadata.duration` для статистики
