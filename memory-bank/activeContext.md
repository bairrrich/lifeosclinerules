## Активные решения и соображения

### Архитектура рецептов
- **Три типа рецептов** — food, drink, cocktail с общими и специфичными полями
- **Отдельные таблицы** — ингредиенты, шаги хранятся отдельно от контента
- **Metadata поля** — food_metadata, drink_metadata, cocktail_metadata для специфичных данных
- **КБЖУ на уровне контента** — calories, protein, fat, carbs, sugar, fiber

### UI паттерны
- Tabs для выбора типа рецепта
- Card секции для логической группировки полей
- Color-coded tabs для разных типов (orange, blue, purple)
- Drag-and-drop для шагов (в планах)
