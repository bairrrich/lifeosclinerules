Для унифицированной базы рецептов (еда / выпечка / десерты / супы / напитки / смузи / коктейли / безалкогольные напитки и т.д.) в 2025–2026 годах наиболее практичная и гибкая структура — **одна основная таблица рецептов + расширяемость через JSONB / атрибуты / отдельные таблицы метрик**.

Вот как обычно разделяют **общие** и **специфические** поля, чтобы база была нормализованной, поиск был быстрым, а UI оставался удобным и понятным.

### Самая важная часть — ингредиенты (отдельная связь many-to-many)

**Таблица recipe_ingredients** (или recipe_lines)

| Поле          | Тип       | Еда   | Напитки / Коктейли | Примечание / UI                                    |
| ------------- | --------- | ----- | ------------------ | -------------------------------------------------- |
| recipe_id     | FK        | да    | да                 | —                                                  |
| ingredient_id | FK        | да    | да                 | ссылка на справочник ингредиентов                  |
| amount        | numeric   | да    | да                 | 50, 1.5, 2                                         |
| unit_id       | FK / enum | да    | да                 | г, мл, шт, щепотка, dash, oz, cl, cup, tbsp…       |
| optional      | boolean   | часто | часто              | «по вкусу», «по желанию»                           |
| note          | varchar   | часто | часто              | «свежий», «сухой», «домашний», «лучше выдержанный» |
| order         | int       | да    | да                 | порядок отображения в UI                           |

**Справочник ingredients** (очень важно сделать хороший)

- id
- name (водка, куриная грудка, банан…)
- name_en (для многоязычности)
- category (spirit, juice, fruit, vegetable, dairy, herb, sweetener…)
- subcategory (белый ром / темный ром / выдержанный ром…)
- is_alcoholic (boolean)
- default_unit (часто мл для жидкостей, г для сыпучих)
- calories_per_100g_or_100ml
- density (для пересчёта объём ↔ масса, если нужно)

### Рекомендуемая структура на 2026 год (самая популярная)

```text
recipes
├── id, title, slug, description
├── type (food / beverage / cocktail)
├── sub_type (breakfast / soup / smoothie / sour / tiki …)
├── difficulty, servings, times (prep/cook/total)
├── is_alcoholic, abv_percent
├── glass_type, method, ice_type, garnish
├── instructions (array или JSON)
├── tags, diets, tools
├── images[], video_url
└── extra_attributes JSONB    ← сюда всё редкое: smoke, carbonation, layering difficulty…

recipe_ingredients (связующая)
├── recipe_id
├── ingredient_id
├── amount, unit_id, optional, note, order
└── substitute_ingredient_id (опционально)

ingredients
├── id, name, name_en, category, subcategory
├── is_alcoholic, default_unit, calories_per_100…
└── extra JSONB (ABV для спиртного, сезонность…)

units (отдельная таблица или enum)
```

### Краткий совет по UI

- **Обязательно** показывать тип/подтип крупно (еда / коктейль / смузи)
- Разные иконки и цветовые акценты для алкогольных / безалкогольных
- Фильтры: алкогольный / безалкогольный / < 10% / без сахара / < 15 мин
- Для коктейлей — отдельный блок «Метод + Лёд + Гарнир + Бокал»
- Ингредиенты всегда в красивом списке с иконками единиц измерения
- Возможность переключения единиц (ml ↔ oz, г ↔ чашки)

Если хочешь — могу предложить конкретный SQL (PostgreSQL) или пример JSON-схемы для одной записи (еда vs коктейль).

Для унификации и нормализации базы данных рецептов (еда, напитки, коктейли) важно выделить общие для всех типов поля и специфические, характерные только для конкретной категории. Это позволит избежать избыточности данных, упростит поддержку и обеспечит гибкость пользовательского интерфейса (динамическое отображение полей в зависимости от выбранного типа рецепта).

## Общие поля (для всех типов рецептов)

Эти поля присутствуют в любом рецепте независимо от категории. Их можно вынести в основную таблицу `recipes`.

| Поле          | Тип                             | Описание                                                                           |
| ------------- | ------------------------------- | ---------------------------------------------------------------------------------- |
| `id`          | INT (PK)                        | Уникальный идентификатор рецепта                                                   |
| `title`       | VARCHAR                         | Название рецепта                                                                   |
| `description` | TEXT                            | Краткое описание (превью)                                                          |
| `author_id`   | INT (FK)                        | Ссылка на пользователя (автора)                                                    |
| `created_at`  | DATETIME                        | Дата создания                                                                      |
| `updated_at`  | DATETIME                        | Дата последнего обновления                                                         |
| `image_main`  | VARCHAR                         | Путь к главному изображению                                                        |
| `gallery`     | JSON                            | Массив дополнительных изображений (опционально)                                    |
| `servings`    | INT                             | Количество порций / выход                                                          |
| `prep_time`   | INT                             | Время подготовки (в минутах)                                                       |
| `cook_time`   | INT                             | Время приготовления (в минутах)                                                    |
| `total_time`  | INT                             | Общее время (вычисляемое или заполняемое)                                          |
| `difficulty`  | ENUM                            | Уровень сложности (легко, средне, сложно)                                          |

## Специфические поля (расширение в зависимости от типа)

Для каждого типа рецепта создаётся отдельная таблица с дополнительными атрибутами. Связь с основной таблицей — один-к-одному через `recipe_id`.

### 1. Для еды (`recipes_food`)

| Поле             | Тип          | Описание                                                                     |
| ---------------- | ------------ | ---------------------------------------------------------------------------- |
| `recipe_id`      | INT (PK/FK)  | Ссылка на рецепт                                                             |
| `cuisine`        | VARCHAR      | Кухня (итальянская, японская и т.п.)                                         |
| `meal_type`      | ENUM         | Тип блюда (завтрак, суп, салат, горячее, десерт, выпечка и т.д.)             |
| `cooking_method` | SET          | Методы приготовления (жарить, варить, запекать и т.д. – множественный выбор) |
| `calories`       | INT          | Калорийность на порцию (ккал)                                                |
| `protein`        | DECIMAL(5,1) | Белки на порцию (г)                                                          |
| `fat`            | DECIMAL(5,1) | Жиры на порцию (г)                                                           |
| `carbs`          | DECIMAL(5,1) | Углеводы на порцию (г)                                                       |
| `dietary_info`   | SET          | Особенности (веганское, безглютеновое, низкоуглеводное и т.п.)               |

### 2. Для напитков (безалкогольные) (`recipes_drink`)

| Поле             | Тип         | Описание                                                                       |
| ---------------- | ----------- | ------------------------------------------------------------------------------ |
| `recipe_id`      | INT (PK/FK) | Ссылка на рецепт                                                               |
| `drink_type`     | ENUM        | Тип напитка (чай, кофе, сок, смузи, лимонад, молочный коктейль, компот и т.д.) |
| `base`           | VARCHAR     | Основа (вода, молоко, сок, чай, кофе и т.п.)                                   |
| `temperature`    | ENUM        | Температура подачи (горячий, тёплый, холодный, со льдом)                       |
| `serving_volume` | INT         | Объём порции (мл)                                                              |
| `with_ice`       | BOOLEAN     | Подаётся со льдом                                                              |
| `additives`      | TEXT        | Дополнительные ингредиенты / украшения (опционально)                           |

### 3. Для коктейлей (алкогольные/безалкогольные миксы) (`recipes_cocktail`)

| Поле              | Тип          | Описание                                            |
| ----------------- | ------------ | --------------------------------------------------- |
| `recipe_id`       | INT (PK/FK)  | Ссылка на рецепт                                    |
| `alcohol_content` | ENUM         | Тип (алкогольный, безалкогольный)                   |
| `strength`        | DECIMAL(3,1) | Крепость (об. %, опционально)                       |
| `base_spirit`     | VARCHAR      | Основной алкоголь (водка, джин, ром, текила и т.п.) |
| `method`          | ENUM         | Способ приготовления (шейк, стир, билд, бленд)      |
| `glass`           | VARCHAR      | Тип бокала (хайбол, олд фешен, мартини и т.д.)      |
| `garnish`         | VARCHAR      | Украшение (долька лимона, оливка, мята)             |
| `ice`             | ENUM         | Тип льда (кубики, дроблёный, краш, без льда)        |
| `layered`         | BOOLEAN      | Слоистый коктейль                                   |
| `color`           | VARCHAR      | Цвет напитка (опционально)                          |

## Нормализация ингредиентов и шагов приготовления

Для всех типов рецептов ингредиенты и шаги лучше хранить унифицированно.

### Ингредиенты

- Таблица `ingredients` — словарь продуктов (название, возможно категория).
- Таблица `units` — словарь единиц измерения (грамм, мл, шт., чайная ложка и т.д.).
- Таблица `recipe_ingredients`:
  - `recipe_id` (FK)
  - `ingredient_id` (FK)
  - `quantity` DECIMAL(8,2)
  - `unit_id` (FK)
  - `note` VARCHAR (например, "по вкусу", "мелко нарезать")

### Шаги приготовления

- Таблица `recipe_steps`:
  - `recipe_id` (FK)
  - `step_number` INT
  - `description` TEXT
  - `image` VARCHAR (опционально)
  - `timer_minutes` INT (время выполнения шага, если важно)

## Категории и теги

Для удобной фильтрации можно создать общую систему категорий и тегов, но с возможностью привязки к конкретному типу рецепта (чтобы в UI показывать только релевантные фильтры).

- Таблица `categories` (`id`, `name`, `type` (food/drink/cocktail/all)) — категории могут быть общими или привязанными к типу.
- Таблица `recipe_category` (`recipe_id`, `category_id`).

Аналогично для тегов.

## Преимущества для UI

1. **Динамические формы** — при создании/редактировании рецепта, выбрав тип, форма подгружает только нужные дополнительные поля.
2. **Фильтры и поиск** — можно точно фильтровать по специфическим атрибутам (например, "коктейли на основе джина" или "веганские блюда").
3. **Детальный просмотр** — на странице рецепта отображаются только те характеристики, которые уместны.
4. **Валидация** — проще проверять корректность заполнения специфических полей.
5. **Масштабирование** — легко добавить новый тип рецепта (например, "выпечка") без изменения основной структуры — просто создаётся новая таблица с дополнительными полями.

Такая структура обеспечивает нормализацию (избегаем множества NULL-полей в одной таблице) и гибкость интерфейса.

> **Базовая сущность Recipe + расширения по типу + нормализованные справочники**

Главный принцип:
**Тип рецепта влияет только на UI и дополнительные поля, но не ломает базовую модель.**
